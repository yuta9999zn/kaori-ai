K-rules (K-1 .. K-20) — Critical Invariants

K-1 RLS multi-tenant CLAUDE.md §4
Every SELECT filters WHERE tenant_id = $1 (hoặc enterprise_id); RLS enforced via acquire_for_tenant GUC. Postgres Row-Level Security policy gắn vào mọi tenant-scoped table. (ADR-0013)
K-2 Bronze append-only CLAUDE.md §4
Bronze tables no UPDATE/DELETE. Snapshot table customer_workflow_versions cũng K-2 (immutable trigger refuse UPDATE/DELETE).
K-3 LLM via llm-gateway only CLAUDE.md §4
Tất cả LLM call qua llm-gateway service — không direct SDK. Cost governance + consent + drift control. Exception documented: PageIndex upstream wrap (ADR-skipped, opt-in per tenant).
K-4 External AI consent CLAUDE.md §4 + ADR-0015
External AI vendor only with consent_external=True per tenant + per-call prefer_external=True. Qwen 2.5 14B local là default cho mọi tenant. data_residency_strict=true override consent — luôn Qwen. OCR + embedding endpoints REFUSE consent_external entirely (schema-level pin via test) — image bytes + raw text mang PII byte-level redaction không strip được; vendor vision/embed = Phase 3 + separate ADR.
K-5 PII redaction CLAUDE.md §4
Email/phone/ID/name → [redacted] trước mọi external API call. Vietnamese-aware (vn_phone normalizer +84/0 prefix, vn_name strip diacritics + đ→d).
K-6 Decision audit log CLAUDE.md §4
Mỗi automated decision INSERT vào decision_audit_log với: model_version + prompt_hash + alternatives evaluated + confidence + reasoning. K-6 + mig 098 ai_decision_audit (Phase 2.7).
K-7 JWT claims X-* headers CLAUDE.md §4
JWT claims forwarded từ gateway sang downstream services qua X-Enterprise-ID / X-User-ID / X-User-Role / X-Tenant-ID headers.
K-8 SHA-256 idempotent pipeline CLAUDE.md §4
File fingerprint = SHA-256(content). Same file = skip duplicate. Bronze unique constraint (tenant_id, sha256).
K-12 tenant_id from JWT only CLAUDE.md §4
Anti-IDOR. tenant_id NEVER accepted via query string / body / header — chỉ JWT claim. Path params validate trùng JWT (e.g. /p2/workflows/{wf_id} wf_id phải thuộc tenant JWT).
K-13 Idempotency-Key on POST CLAUDE.md §4 + ADR-0014
POST mutations require Idempotency-Key header. Redis 24h API-layer + Postgres idempotency_records 7d per-node. Workflow re-run safe.
K-17 side_effect_class CLAUDE.md §4 + ADR-0014
Every workflow node declare side_effect_class ∈ {pure, read_only, write_idempotent, write_non_idempotent, external}. Ép retry policy + saga compensation đúng. Test mode (P2-27) gate external nodes phải mock-able.
K-18 Vault-only secrets prod CLAUDE.md §4
Phase 1.5+ production profile: Vault là secret store duy nhất. Dev/staging cho env-var fallback (log warning). KAORI_PROFILE=production refuse env fallback. Field-key + MFA master key đều resolve from Vault.
K-19 OTel tenant_id span CLAUDE.md §4
OpenTelemetry mandatory; mọi span MUST carry attribute tenant_id. Cross-tenant trace search + leak detection.
K-20 LLM version pinning CLAUDE.md §4 + ADR-0015
Per-workflow pin model: claude-sonnet-4-6, version: "2026-01-01". No silent vendor upgrade. Drift control + reproducibility.

Workflow Card (7-field SSOT)

Workflow Card Chốt 2026-05-20 · workflow-builder-ux.html § Step 3-4
Đơn vị công việc trên canvas (= node trong DB). SSOT cho card format ở workflow-builder-ux.html.
2-layer field model (chốt Round 3 2026-05-21 sau review):
Card-level 7-field (per-node, render trong Builder editor mockup):
  1. Owner — người chịu trách nhiệm card này (role hoặc user)
  2. Input — variables / artifacts từ card trước
  3. Required docs — tài liệu phải upload trước khi card chạy (link mig 053 step_documents)
  4. AI action — LLM call (read_only) hoặc compute action
  5. Branch — rẽ nhánh đầu ra theo condition
  6. Output — variables card này produce cho card kế
  7. SLA — time-to-complete + escalation rule
Workflow-level fields (per-workflow, render trong Catalog Mermaid + summary table):
  • Trigger — event/schedule/webhook bắt đầu workflow (e.g. cron / form_submission / webhook / manual)
  • SLA tổng — end-to-end time-to-complete (sum/max của card SLAs + branch wait time)
  • Owner workflow — dept owner (Marketing/Sales/CS/Warehouse/Finance/System), KHÔNG phải owner per-card
  • industry_vertical — applicability tag (general/saas/fintech/retail/manufacturing/logistics)
  • Mode + Plan gate — Simple/Advanced/Developer + PILOT/ENT_BASIC/ENT_MID/ENT_MAX
  • Executable status — LIVE / PARTIAL / SEED ONLY
Render mapping:
• Catalog view (feature-workflows.html) = workflow-level fields trong Mermaid flowchart + steps-table per node. KHÔNG render 7-field per card (40 workflows × 5 cards avg = 200 cards quá dày).
• Editor view (workflow-builder-ux.html) = 7-field per card trong Card Stack vertical + drawer editor. 1 workflow tại 1 thời điểm.
• Summary table (§ Mode + Plan Gate matrix) = workflow-level columns 40 rows.

side_effect_class taxonomy (K-17)

pure
No side effect, no external read. Pure compute. Idempotent infinitely. Example: if_else, switch, aggregate, filter, transform, dedup_records.
read_only
Read DB / external API, no write. Cacheable. Example: read_table, read_form_submission, classify_text, generate_narrative, rag_query, AI nodes wrapping llm-gateway.
write_idempotent
Write DB với UNIQUE constraint hoặc UPSERT. Safe re-run. Example: update_record, save_to_database, create_task (UNIQUE task_key).
write_non_idempotent
Write DB không idempotent. Needs saga compensation cho rollback. Example: balance debit, sequential ID increment.
external
External API call (HTTP / SMTP / webhook). Side effect bên ngoài hệ thống. Needs idempotency-key + saga compensation handler. Example: send_email, send_chat_message, call_api, send_sms.

8-Step Silver Pipeline (Round 5 N3)

Silver transformation pipeline SRS §2 · MEDALLION_CONTRACT.md §Silver Pipeline · Round 5 N3
Canonical 8 bước transformation Bronze→Silver, mỗi row MUST pass in order:
  1. Schema validation — Required columns + enum values (silver/schema_validator.py)
  2. Type cast — Source string → NUMERIC/BIGINT/DATE/BOOLEAN; coerce-fail flagged not dropped (silver/type_coercer.py)
  3. Null handling — Drop OR impute per rule (median/mean/forward-fill/business default) (silver/null_handler.py)
  4. Dedup — SHA-256 row fingerprint per business key; VN-aware (P2.5 dedup_records mig 086)
  5. PII masking VN — Tên→{{role:X}}; SĐT→{{phone:****1234}}; CCCD→{{cccd:hash:abc123}}; email→hash; address→region only. NFR-PR-01 F1 ≥0.95
  6. Normalize — SĐT +84 E.164; tiền VND integer; ngày ISO 8601; tên công ty canonical; địa chỉ 4-level (silver/normalizer.py)
  7. Outlier flag — Z-score (>3σ) + IQR (>1.5×) tag rows; KHÔNG drop (silver/outlier_detector.py)
  8. Lineage tag — bronze_file_id FK + pipeline_run_id + silver_pipeline_version + transformations_applied JSONB; mig 097 data_lineage_edges
Gate ≥80%: Weighted avg 7-dim quality scorecard ≥ tenant target trước promote Gold. KHÔNG auto-skip.

A. Palette — toàn bộ ký hiệu theo nhóm

B. Ví dụ chuẩn — "Duyệt 2 cấp" (tái hiện đúng ảnh của anh)

3 pool/lane theo role: Người duyệt 1 · Process Engine (hệ thống) · Người duyệt 2. Mỗi cấp: nhận yêu cầu (message start ✉) → User Task "đánh giá" → "lập & gửi quyết định" → end. Engine: User Task "quyết định duyệt" → exclusive gateway "đã duyệt?" yes/no. Message flow (đứt) nối engine ↔ người duyệt; sequence flow (liền) trong pool.

Liền = Sequence Flow (trong 1 pool). Đứt + mũi tên rỗng = Message Flow (giữa 2 pool). Vòng đậm = End. ✉ = Message Start. Người = User Task. ◇✕ = Exclusive Gateway.

C. Quy trình dựng: từ yêu cầu khách → pool/lane/nghiệp vụ

anh nhấn mạnh: workflow phải chia rõ pool & nghiệp vụ ngay từ yêu cầu người dùng. Đề xuất luồng dựng builder:

  1. Nhập yêu cầu nghiệp vụ (ngôn ngữ thường) — VD "Đơn giảm giá cần qua 2 cấp duyệt".
  2. Xác định participant → Pool/Lane: ai tham gia? (Khách, Người duyệt 1/2, Hệ thống). Mỗi role = 1 lane; bên ngoài tổ chức = pool riêng (giao tiếp bằng message flow).
  3. Liệt kê nghiệp vụ → Task trong từng lane, chọn đúng loại: User Task (người làm), Service Task (hệ thống tự chạy), Send/Receive (gửi/nhận), Business Rule (luật quyết định).
  4. Điểm rẽ → Gateway: Exclusive (1 nhánh), Parallel (chạy song song), Inclusive (nhiều nhánh có điều kiện), Event-based (chờ sự kiện nào tới trước).
  5. Sự kiện đầu/cuối → Event: Start (None/Message/Timer…), End (None/Message/Terminate…), boundary cho lỗi/timeout.
  6. Nối: sequence flow trong pool; message flow giữa pool; gắn điều kiện lên nhánh gateway.
Lưu trữ & kéo-thả: mỗi ký hiệu ở §A là 1 entry trong catalog (bpmn-elements.ts) với id · category · bpmnType · marker · icon. Palette kéo-thả tương lai render trực tiếp từ catalog này — không hardcode.

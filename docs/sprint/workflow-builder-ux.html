Reference — ADVANCED mode layout (so sánh)

Khi tenant chuyển sang Advanced mode (qua nút "⚙ Chuyển sang Advanced để sửa" ở trên hoặc PATCH /enterprises/{id}/workflow-mode), cùng card editor này thay đổi:

SIMPLE (hiện tại) SIMPLE
  • ✏ Edit: Tên card / Owner / SLA / Required docs / Threshold cảnh báo
  • 🔒 Disabled: Input · AI action · Branch · Output (chỉ xem)
  • Branch row có 2 button rõ: 🔍 Xem logic nhánh + ⚙ Chuyển sang Advanced
  • Save → workflow_customizations với edit_mode='simple' + snapshot customer_workflow_versions
  • Role tối thiểu: OPERATOR
ADVANCED (sau khi chuyển) ADVANCED
  • ✏ Unlock: Input schema · AI prompt edit · Branch add/remove/edit · Output shape
  • Branch Inspector panel mở (right pane): xem + sửa if/then/else/fallback với test sample data button
  • Mode badge đổi sang xanh dương; chip ADV trên field nay là edit-icon
  • Sau ≥5 customization HOẶC schema change → system suggest "📨 Mở Change Request chính thức" (BA 4.2)
  • Role tối thiểu: ANALYST hoặc MANAGER+; plan-gated ENT_BASIC trở lên
  • Save → workflow_customizations với edit_mode='advanced' + bắt buộc test run trước publish

Developer mode không demo ở đây — Developer = canvas drag-drop 45 node-type catalog + YAML export/import + K-17 side_effect_class exposed + Saga compensation + custom node type. Plan-gated qua hợp đồng Custom Implementation. Xem feature-screens.html § P2-26 Workflow Builder cho 3-mode spec đầy đủ.

BE mapping (cho dev / AI agent):
Step 1 (Template Library) → GET /industries/{id}/workflows?recommendation_level=core + per-dept filter từ industry_workflow_links (mig 101) joined với workflow_templates (mig 054).  |  Step 2 (Template Detail) → GET /workflow-templates/{tid} + GET /industries/{id}/departments + GET /industries/{id}/kpis?dept_key=sales + GET /industries/{id}/data-schemas + GET /workflow-templates/{tid}/cross-links (mig 057).  |  Step 3 (Card Stack) → POST /workflows/instantiate-from-template (lần đầu mở; clone template thành tenant-owned workflow row mig 053) + GET /workflows/{wf_id}/nodes.  |  Step 4 (Card Editor) → POST /workflows/{wf_id}/customize với operation=edit_node, edit_mode=simple|advanced → Save → INSERT workflow_customizations + snapshot customer_workflow_versions mới (K-2 immutable trigger).

K-rule enforcement:
  • K-1 RLS — mọi customer_workflow_* + workflow_customizations tự động lọc theo tenant_id từ JWT.
  • K-2 immutable — snapshot customer_workflow_versions bị trigger refuse UPDATE/DELETE; mỗi save = row mới.
  • K-12 anti-IDOR — path {wf_id} phải thuộc tenant trong header X-Enterprise-ID; mismatch → 403 RFC 7807.
  • K-13 idempotency — POST customize cần header Idempotency-Key (Redis TTL 24h API + Postgres idempotency_records TTL 7d node).
  • K-17 side_effect_class — Simple mode chỉ exposed UI; Advanced mode hiển thị; Developer mode edit-able.
  • K-19 OTel — mọi span ghi tenant_id + workflow_id + customize.edit_mode attribute.

Mode gating: SIMPLE = SME inline edit (OPERATOR+); ADVANCED = analyst (ANALYST/MANAGER+, plan ENT_BASIC+); DEVELOPER = platform admin (Custom Implementation contract). Field disabled trong Simple sẽ unlock khi tenant PATCH /enterprises/{id}/workflow-mode sang advanced. Toggle mode trên header workflow (Step 3) chỉ enable button tương ứng với plan + role hiện tại.

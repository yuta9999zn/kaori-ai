Kaori AI là SaaS B2B (+ một dòng Personal nhỏ) biến dữ liệu kinh doanh thành quyết định — không cần data engineer. v4 mở rộng qua 24 tháng × 36 sprint × ~1147 features, phục vụ 4 audience × 6 portal:

10 nguyên tắc kiến trúc (lấy từ SAD_SKELETON_V2.md)

  1. Layered architecture — 6 layer rõ ràng, dependency một chiều top-down request, bottom-up data.
  2. Event-driven backbone — Redis Streams Phase 1 + Kafka topic legacy; async by default.
  3. Polyglot persistence — Postgres + pgvector (transactional + Phase 1-2 vector embedded) + ClickHouse (analytical) + MinIO (blob) + Redis (cache) + Vault (secrets); external managed vector DB (Pinecone / Qdrant) cân nhắc Phase 3 cho data-residency-strict workload.
  4. Multi-tenancy first-class — every layer carries tenant_id; RLS at DB; cross-tenant leak = critical bug (K-1, K-12).
  5. Reliability over performance — idempotency by design (K-8, K-13), saga compensation, at-least-once + idempotency >>> exactly-once.
  6. AI as layer, not feature — Reasoning Layer là service riêng; LLM version pinned per workflow (K-20).
  7. Process Mining = moat — discover before build; AI mine logs → builder pre-populated.
  8. Operational economics = manager language — mọi capability quy ra VND; NOV monthly; ROI first-class.
  9. Adoption intelligence — track resistance signals (abandonment, override, side-channel); proactive intervention.
  10. Vietnam-native — hosted in Vietnam (data residency); Zalo + Misa + Fast + Bravo first-class; pricing in VND.

Self-discovery ở Silver (CR-0016, 2026-05-27) — không hardcode

Bước 2 (type_cast) + bước 6 (normalize) đo từ chính dữ liệu thay vì giả định locale; phần diễn giải (QUYẾT) được surface cho người dùng/LLM duyệt, không tự áp:

  • Đơn giá vs thành tiền: thêm canonical unit_price (tách khỏi amount); rule DERIVE_LINE_TOTAL (safe=false → user duyệt) tính amount = unit_price × quantity khi chưa có total; measure_amount_signals ĐO bằng chứng để quyết. Gold không đổi (normalize ở Silver, Gold strict).

5.4 Chat tool-calling guardrails (Phase 2.8 Round 5)

  • Knowing-doing gap heuristic (ADR-0023) — keyword-score user message hop 0 → force tool_choice="required" nếu score ≥ 0.7. Closes ~80% paper-measured gap.
  • Depth/width loop guardrail (ADR-0023 Alt 5, DPEPO-borrow 2026-05-21) — track tool-call history per session, abort khi combined_penalty < 0.3. Penalty = α^depth × ω^width với α=0.7, ω=0.8. Refuses dispatch + feeds structured error back to LLM.
  • Tool necessity gate by JWT (K-15, K-16) — mỗi tool call kiểm authz per tenant_id + audit log; tools never accept tenant_id/user_id/workspace_id from arguments (JWT-only via ToolContext).

5.5 Reasoning substrate — memory trust + foundational knowledge (2026-05, NNL-Harness)

Mang 3 năng lực từ harness cục bộ NNL vào Kaori, multi-tenant + additive (engine không đổi); tri thức từ DB, ngưỡng env-configurable (không hardcode).

  • Memory trust + palace (ADR-0030 / 0032, mig 110): mỗi ký ức Stage 7 có độ-tin suy giảm theo tuổi trust = confidence × 0.5^(age/half-life) + cờ "confident-but-unchecked"; verify()/reinforce() tươi lại + leo confidence (học theo lần dùng); experience_level = "càng nhiều tháng càng biết nhiều"; recall đi 1-hop theo links (associative). Tách bạch với importance (giữ/quên). Knobs KAORI_MEM_*.
  • Foundational knowledge base (ADR-0033, mig 106/107/111): authority tier 1–4 (regulatory > curated > market > tenant); lớp nền (tier 1/2) bền + chín theo tuổi (reinforce-on-cite, ceiling per tier); version-history (supersedes + change_reason) trả lời "vì sao lại vậy"; lớp biến-động (tier 3) refresh theo chu kỳ.
  • CDFL |OR| coverage gate — "học 1 hiểu 10": knowledge_coverage trên tri thức nền → đủ phủ thì khái quát hoá tự tin (kèm "độ phủ tri thức nền X%"), thiếu thì "kiến thức nền chưa đủ" — KHÔNG bịa (K-3). Inject doc nền vào narrative phân tích thay cho principles hardcode. Knobs KAORI_KB_*.

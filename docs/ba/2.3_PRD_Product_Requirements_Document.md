## 1. Mục đích & phạm vi

PRD là bản lề giữa Stakeholder Requirement và Solution Requirement: mô tả **sản phẩm phải là gì** (capabilities, hành vi, ràng buộc) ở mức đủ để Engineering thiết kế. PRD này **không** lặp lại business context (xem BRD) hay liệt kê toàn bộ feature (xem Feature Tree v4.1) — tập trung vào kiến trúc sản phẩm, năng lực cốt lõi, và 4-phase scope.

---

## 2. Tổng quan sản phẩm

Kaori AI = hệ điều hành ra quyết định cho SME, vận hành theo 7 cơ chế: data normalization → workflow observability (qua Process Mining) → AI-assisted decisioning → human-in-loop → organizational learning loop → privacy-first LLM → value-based monetization (NOV).

**6 portal:** P1 Platform Manager · P2 Enterprise (lõi sản phẩm) · P3 Studio · P4 Personal · P5 Shared Infrastructure · P6 Billing. **~67 module / ~1.147 feature** (Feature Tree v4.1 + Round 4-5 CS vertical delta — 5 màn P2-33..37 per CR-0011).

**Khác biệt v4.1 so với v3.0:** 5 năng lực moat mới — **Process Mining (EPIC-12)** auto-discovery workflow, **Adoption Intelligence (EPIC-13)** 9 signal, **NOV Engine (EPIC-14)** ROI bằng VND, Runtime Reliability, Observability — và tách Phase 1.5 Stabilization.

## 3. Kiến trúc sản phẩm — 6 lớp (SAD layers)

| Lớp | Tên | Năng lực chính |
|---|---|---|
| L0 | Infra | K8s VN region, Postgres + ClickHouse + MinIO + Redis + Vault |
| L1 | Enterprise Integration | API Gateway (JWT/RBAC, tenant routing, rate limit), Ingest, File Upload, Event Stream, 10+ connector |
| L2 | Data & Feature | **Medallion 3 lớp** (Bronze raw → **Silver cleaned/standardized** → Gold business-ready), Feature Store offline+online, Drift Monitor, MLflow |
| L3 | AI/ML Pipeline | Training pipeline, Model Registry, Model Serving (vLLM Qwen), Continuous Learning |
| L4 | Decision Intelligence | Decision Engine, Workflow Engine (45 node executor + 25 production template LIVE), Agent System (Phase 2-3), Serving API |
| L4.5 | **Workflow Intelligence (v4.1 moat)** | **Process Mining (EPIC-12), Adoption Intelligence (EPIC-13), NOV Engine (EPIC-14)** |
| L5 | Explainability & Governance | Explainability (SHAP), Fairness, Audit Log bất biến, Ontology Governance (Phase 2.6), Lineage Edges (Phase 2.7) |
| L6 | Reliability & Ops | Kafka + DLQ + retry + idempotency ledger, OpenTelemetry, DR multi-region, Chaos testing |

> **Medallion — định nghĩa 3 lớp:**
> - **Bronze (Đồng) — raw:** lưu nguyên trạng + envelope (tenant_id, ingest_ts, source, checksum SHA-256). **Không transform**.
> - **Silver (Bạc) — cleaned & standardized:** schema validation, type cast, null handling, **dedup**, **PII masking tiếng Việt**, normalize (SĐT → E.164, tiền tệ → VND), gắn **lineage tag**. Là *interface contract* nội bộ: mọi dữ liệu sau Silver đều cùng schema chuẩn của Kaori. **Quality gate ≥80% records pass mới được lên Gold.**
> - **Gold (Vàng) — business-ready:** aggregate + feature engineering + join đa nguồn + denormalize cho dashboard/insight; là đầu vào cho Insights Engine, Report, NOV.

## 4. Năng lực cốt lõi (Product Capabilities)

### 4.1 Data Foundation (Phase 1) — EPIC-01
Pipeline Wizard 5 bước (Upload → Columns → Clean → Analyze → Results); Auto Schema Detection (Qwen phân tích 100 dòng); Auto Database Design (3NF/Star + ERD + form); Medallion engine; Data Quality Scorecard gợi ý cleaning tiếng Việt.

### 4.2 Insights & Decision (Phase 1) — EPIC-02
Insights Engine 3-tuyến ("Chuyện gì / Tại sao / Nên làm gì") + citation + confidence + calibration; Multi-tier Analysis; Analysis Frameworks; AI Decision Log bất biến + Override + feedback retrain; Explainability SHAP top-3 dịch business tiếng Việt.

### 4.3 Process Mining ⭐ (Phase 1, moat) — EPIC-12
Khám phá workflow thực tế từ event log:
- **Phase 1 (4 nguồn):** Postgres CDC, Excel revision, Zalo metadata, Gmail thread
- **Phase 2 (10+ nguồn):** thêm Outlook, Calendar, Slack, Teams, SharePoint, webhook
- PII tiếng Việt redact → role TRƯỚC khi mining
- Case inference + Heuristic miner (Phase 1) → Inductive + Fuzzy (Phase 2)
- Findings: bottleneck, shadow process, bypass risk
- **Translate to Builder** sinh workflow DRAFT

**Đây là cách Phase 1 đưa workflow vào hệ thống — không phải kéo-thả thủ công.**

### 4.4 Adoption Intelligence ⭐ (Phase 1/1.5, moat) — EPIC-13
**9 tín hiệu kháng cự:**
1. workflow abandonment
2. AI override rate >40%
3. side-channel Zalo/Excel
4. workaround file (parallel-excel — endpoint `GET /adoption/signals/workaround-file-creation-(parallel-excel)`)
5. manager intervention freq
6. completion rate
7. negative sentiment
8. time-on-task variance
9. usage decline (Phase 1.5)

Composite health score 0–100 per workflow/dept/tenant; phân loại 5-tier EXCELLENT/HEALTHY/AT_RISK/STRUGGLING/CRITICAL. Intervention playbook + effectiveness tracking (Phase 1.5).

### 4.5 NOV Engine ⭐ (Phase 1, moat) — EPIC-14
**NOV = Net Operating Value = revenue − cost (VND)**, per workflow per tenant, cron ngày 1 hàng tháng.
- **Revenue:** Pre/Post (Phase 1) + A/B (Phase 1.5) + industry benchmark + KPI-to-revenue mapper + confidence score
- **Cost:** people (time saved × rate) + infra + AI token + integration + opportunity + setup amortization
- **Output:** NOV monthly, time-to-payback, variance (predicted vs actual)
- **Đây là ngôn ngữ ROI cho quản lý** — thay framing trừu tượng cũ.

### 4.6 Multi-tenant & AuthZ (Phase 1 / Phase 2) — EPIC-03
Row-level isolation fail-closed; RBAC Phase 1; ABAC + Hybrid PDP Phase 2; Audit Log bất biến retention 2 năm.

### 4.7 Privacy-first LLM (Phase 1 core / Phase 2 full) — EPIC-05
Qwen 2.5 (7B/14B/32B) via vLLM nội bộ mặc định; LLM Router theo privacy_mode/PII/task; External Gateway chỉ nhận data masked; Guardrails Phase 2; Agent Framework Phase 2-3.

### 4.8 Workflow Automation (Phase 2) — EPIC-08
Workflow Builder drag-drop canvas, config panel/step, auto-versioning, rollback 30 ngày, trigger đa loại, action library (45 node executor LIVE: email/webhook/CRM/Slack/SMS/AI nodes/document); 90-day parallel test (DRAFT→REVIEWING→TESTING→ACTIVE) với auto-rollback; Knowledge Graph; Blast Radius; Multi-user collab (workflow_editors/comments/locks Phase 2.5); Approval Gates; Workflow Events (event sourcing Phase 2.6); Idempotency Ledger (Phase 2.6).

- **Security:** JWT RS256, AES-256-GCM at-rest, TLS 1.3 in-transit, MFA bắt buộc Admin, PII masking trước external LLM.
- **Compliance:** GDPR/PDPA Phase 1; e-invoice Nghị định 123 Phase 1; SOC2/ISO27001/EU AI Act Phase 3.

### 6.1 Product Behavior chuẩn (bắt buộc mọi screen) — MỚI

Mọi screen sản phẩm phải hỗ trợ **4 state** (NFR-U-06):

| State | Mô tả | Ví dụ implementation |
|---|---|---|
| **Empty** | Khách chưa có data | CTA "Upload dữ liệu đầu tiên" + link wizard; không phải dashboard trắng |
| **Loading** | Đang tải/compute | Skeleton screen; spinner sau 500ms; cancel button sau 10s |
| **Error** | Lỗi (4xx/5xx) | Error message tiếng Việt cụ thể; nút "Thử lại"; link liên hệ support |
| **Permission denied** | Không đủ quyền | "Bạn không có quyền truy cập tính năng này. Liên hệ Manager." + role hiện tại + role cần |

### 6.2 Audit Action chuẩn (bắt buộc mọi action có write)

Mọi action ghi DB phải emit audit event với: `actor_user_id`, `tenant_id`, `action_type`, `resource_id`, `before_state`, `after_state`, `timestamp`, `ip`, `user_agent`. Retention 2 năm WORM storage.

## 1. Mục đích

Đăng ký mọi yêu cầu thay đổi (CR) so với scope đã baseline. Mỗi CR đi qua **quy trình 5 bước** (4.1 §4.2) và được lưu ở đây làm nguồn sự thật.

---

## 2. Quy trình Change Request — 5 bước

```
1. SUBMIT  →  2. ESTIMATE  →  3. ASSESS  →  4. APPROVE  →  5. BASELINE
   │            │              │              │              │
   PM          PO+TL          PM+TL+SEC      PO(L/M)         PO
                                             SP(H/C)
```

**Phân loại CR theo impact:**

| Loại | Tiêu chí | Approver | Thời gian xử lý mục tiêu |
|---|---|---|---|
| LOW | Sửa text, đổi label, fix bug nhỏ, không đổi schema | PO | 3 ngày |
| MEDIUM | Thêm field/feature trong module hiện có, không đổi data model lớn | PO + PM | 1 tuần |
| HIGH | Thêm module mới, đổi data model, đổi phase, ảnh hưởng giá | Sponsor (R: PO) | 2 tuần |
| CRITICAL | Thay đổi kiến trúc, scope phase, pricing model, compliance | Sponsor (R: PO + BOD informed) | 1 tháng |

**Status flow:** `SUBMITTED → ESTIMATING → ASSESSING → APPROVED/REJECTED → IMPLEMENTING → IMPLEMENTED → BASELINED`

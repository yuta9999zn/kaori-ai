## 1. Mục đích

URD trả lời **"Người dùng cần làm được gì?"** — viết bằng ngôn ngữ người dùng, không phải ngôn ngữ kỹ thuật. ### UR-A — Onboarding & Workspace (Journey A) · Phase 1 [AS-IS]

#### US-A1 — Platform Admin tạo workspace + sinh key
> Là **Platform Admin (PLT-02)**, tôi muốn tạo workspace cho doanh nghiệp mới và sinh key KAORI-XXXX, để khách có thể truy cập độc lập.

**AC — Given/When/Then:**
- **Happy path:**
  *Given* tôi đã login với role SUPER_ADMIN hoặc ADMIN,
  *When* tôi submit form tạo workspace với company name + plan + sponsor contact,
  *Then* hệ thống tạo `tenant_id` UUID, sinh key `KAORI-XXXX-XXXX-XXXX` hash SHA-256 lưu DB, raw key hiện 1 lần để copy; email + SMS/Zalo gửi key cho sponsor; audit event `workspace.created` ghi với actor + tenant_id + timestamp.
- **Validation failed:**
  *Given* company name có ký tự đặc biệt không hợp lệ hoặc plan không tồn tại,
  *When* submit,
  *Then* trả lỗi 400 tiếng Việt cụ thể "Tên công ty chỉ chứa chữ/số/khoảng trắng" hoặc "Plan không hợp lệ"; không tạo workspace; ghi audit `workspace.create_failed`.
- **Permission denied:**
  *Given* user login với role SUPPORT (không phải SUPER_ADMIN/ADMIN),
  *When* gọi API tạo workspace,
  *Then* trả 403 + audit `workspace.create_denied` + alert SEC nếu lặp >3 lần/giờ.
- **Dependency unavailable:**
  *Given* SMS provider hoặc email service down,
  *When* sinh key xong,
  *Then* workspace vẫn được tạo; key lưu trong "pending delivery queue"; retry exponential backoff (5 lần); CSM được notify qua Slack để gửi key manual.

#### US-A2 — Enterprise Manager kích hoạt workspace
> Là **Enterprise Manager (ENT-01)**, tôi muốn dùng key kích hoạt workspace và đặt mật khẩu, để bắt đầu sử dụng.

**AC:**
- **Happy:** *Given* tôi nhận key qua email + SMS, *When* nhập key vào trang activate + đặt password ≥12 ký tự, *Then* tạo user MANAGER, key đánh dấu `used_at`, redirect onboarding wizard; key không dùng lại được.
- **Validation:** *Given* key sai format hoặc đã expired (>7 ngày), *When* submit, *Then* "Key không hợp lệ hoặc đã hết hạn. Liên hệ CSM."
- **Permission:** *Given* key đã `used_at` (lần 1), *When* dùng lại key, *Then* 403 "Key đã được sử dụng" + log `key_reuse_attempt` + alert.
- **Dependency:** *Given* DB write fail khi tạo user, *When* commit, *Then* rollback transaction; key chưa đánh dấu used; user thấy "Lỗi tạm thời, thử lại sau 1 phút".

#### US-A3 — Khai báo công ty + mời member đầu tiên
> Là **ENT-01**, tôi muốn khai báo thông tin công ty và mời thành viên đầu tiên trong wizard onboarding, để team cùng vào việc.

**AC:**
- **Happy:** *Given* đã activate workspace, *When* hoàn thành wizard (company name, industry, size, address, ≥1 invite email), *Then* lưu company profile; gửi invite email với token 72h; enforce ≥1 MANAGER active.
- **Validation:** *Given* email invite format sai, *When* submit, *Then* highlight field + "Email không hợp lệ".
- **Permission:** *Given* user role không phải MANAGER, *When* mời member, *Then* 403; chỉ MANAGER mới mời.
- **Dependency:** *Given* email service down, *When* gửi invite, *Then* invite lưu pending; retry 3 lần; nếu fail → hiện "Sao chép link mời thủ công" với token URL.

#### US-A4 — IT Admin quản lý member với multi-role
> Là **ENT-06 (IT Admin)**, tôi muốn quản lý member (mời/đổi role/deactivate) với multi-role per user, để phân quyền đúng.

**AC:**
- **Happy:** *Given* tôi là MANAGER, *When* gán role [OPERATOR, ANALYST] cho 1 user, *Then* user có cả 2 role; audit `role.granted`; user thấy quyền mới ngay login kế.
- **Validation:** *Given* gán role không tồn tại hoặc role cấm combo (vd VIEWER + MANAGER), *When* submit, *Then* 400 "Role combination không hợp lệ".
- **Permission:** *Given* user thường (không MANAGER), *When* đổi role member khác, *Then* 403 + audit.
- **Dependency (last manager protection):** *Given* chỉ còn 1 MANAGER active, *When* deactivate hoặc remove role MANAGER của user đó, *Then* 409 "Phải có ≥1 MANAGER active" + chặn action.

#### US-A5 — Tập đoàn đa cấp (GAP-01 BE-SHIPPED)
> Là **ENT-01 của một tập đoàn**, tôi muốn **kéo-thả dựng sơ đồ phả hệ tổ chức đa cấp** (tập đoàn → mảng → công ty con → phòng ban) và click vào phòng ban để thao tác, để quản lý cấu trúc nhiều công ty con.

**Status:** 🟡 BE shipped (corporate_tree + cross-link endpoint ở code repo), FE canvas pending.

**AC (when FE shipped):**
- **Happy:** *Given* tenant root đã tạo, *When* tôi kéo node con vào parent trên canvas org-chart, *Then* tạo `org_node` với `parent_id` FK self; KPI/NOV rollup từ leaf → root tự động trong ≤2s; audit `org.node_created`.
- **Validation:** *Given* tạo vòng (A là parent của B, B là parent của A), *When* submit, *Then* 400 "Vòng tròn không cho phép".
- **Permission:** *Given* user role OPERATOR (không phải MANAGER), *When* edit cấu trúc tổ chức, *Then* 403; chỉ MANAGER trở lên mới edit.
- **Dependency:** *Given* tenant chưa có root node, *When* tạo node con đầu tiên, *Then* tự tạo root từ company profile; user không phải làm gì thêm.

#### US-B2 — Auto Schema Detection
> Là **ENT-03**, tôi muốn hệ thống tự nhận diện ý nghĩa cột và map về field chuẩn theo ngành, để không phải khai báo thủ công.

**AC:**
- **Happy:** *Given* Bronze có 100+ dòng, *When* bấm "Detect schema", *Then* Qwen phân tích 100 dòng đầu, gợi ý mapping (vd "ma_kh" → `customer_id`); cho phép sửa từng field; chặn nếu thiếu cột ESSENTIAL của domain.
- **Validation:** *Given* file chỉ có 50 dòng, *When* detect, *Then* warning "Cần ≥100 dòng để detect chính xác" + cho phép skip nếu user chấp nhận risk.
- **Permission:** *Given* user VIEWER, *When* sửa mapping, *Then* 403.
- **Dependency:** *Given* Qwen service P99 >5s timeout, *When* detect, *Then* fallback rule-based detection + warning "AI tạm thời chậm, dùng detect cơ bản".

#### US-B3 — Cleaning rules với preview Before/After
> Là **ENT-03**, tôi muốn xem AI gợi ý luật làm sạch (tiếng Việt) và duyệt từng luật với preview Before/After, để kiểm soát chất lượng.

**AC:**
- **Happy:** *Given* schema confirmed, *When* AI gợi ý luật (dedup, PII mask, normalize SĐT, etc.), *Then* mỗi luật có [Apply]/[Skip]/[Edit]; preview diff 10 dòng sample; sau Apply, **Silver gate ≥80%** records pass; nếu <80% pipeline DỪNG ở Silver, hiện scorecard.
- **Validation:** *Given* luật custom có regex sai, *When* test, *Then* "Regex không hợp lệ tại ký tự thứ X" tiếng Việt.
- **Permission:** *Given* user OPERATOR (không ANALYST), *When* edit luật cleaning custom, *Then* 403; OPERATOR chỉ Apply/Skip.
- **Dependency:** *Given* preview engine fail, *When* xem Before/After, *Then* hiện "Preview tạm thời không khả dụng, Apply vẫn an toàn vì có rollback 7 ngày".

#### US-B4 — Chọn analysis template → Gold
> Là **ENT-03**, tôi muốn chọn template phân tích và chạy ra Gold layer, để có dữ liệu sẵn sàng cho insight.

**AC:**
- **Happy:** *Given* Silver pass ≥80%, *When* chọn template (Churn / RFM / Cohort) + bấm Run, *Then* sinh `analysis_run_id`, Gold view refresh; ETA hiển thị progress bar; xong → notification + link Results.
- **Validation:** *Given* Silver chưa đạt 80%, *When* Run, *Then* "Cần làm sạch thêm: 234 dòng còn lỗi" + jump back Step 3.
- **Permission:** *Given* user VIEWER, *When* Run, *Then* 403.
- **Dependency:** *Given* compute resource saturated, *When* Run, *Then* queue + ETA estimate; nếu queue >30 phút → cảnh báo + đề xuất upgrade.

#### US-B5 — Dashboard + Export
> Là **ENT-04**, tôi muốn xem dashboard kết quả và export, để dùng ngay.

**AC:**
- **Happy:** *Given* Gold đã refresh, *When* mở Results, *Then* dashboard load <3s, có 5 chart mặc định + bảng KPI; nút export PDF/Excel theo plan.
- **Validation:** *Given* Gold chưa có data (analysis fail), *When* mở, *Then* empty state "Phân tích chưa hoàn tất" + link rerun.
- **Permission:** *Given* user dept khác, *When* xem dashboard dept không thuộc, *Then* 403 dept-level isolation.
- **Dependency:** *Given* PDF service down, *When* export, *Then* "Export PDF tạm chưa khả dụng, dùng Excel" + button Excel.

#### US-B6 — "Explain this" → Insight 3-tuyến
> Là **ENT-04**, tôi muốn bấm "Explain this" để nhận insight 3-tuyến, để hiểu *chuyện gì / tại sao / nên làm gì*.

**AC:**
- **Happy:** *Given* mở dashboard có metric bất thường, *When* bấm "Explain", *Then* sinh insight 3-tuyến trong <15s với confidence + citation (top-3 SHAP factor dịch tiếng Việt) + disclaimer "AI-generated, verify before action".
- **Validation:** *Given* metric không có data đủ (n<30), *When* Explain, *Then* "Không đủ data để giải thích đáng tin cậy (cần ≥30 sample)".
- **Permission:** *Given* user VIEWER, *When* override insight, *Then* 403; VIEWER chỉ đọc, ANALYST+ mới override.
- **Dependency (calibration):** *Given* ECE >0.10 (over-confident model), *When* sinh insight với conf 0.85, *Then* hiện "OVER-CONFIDENT WARNING" + bắt buộc manual verify trước khi action.
